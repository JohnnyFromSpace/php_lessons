Порт являет собой цифровое дополнение к ip-адресу. Он предназначается для распределения различных пакетов данных всевозможных служб. Это своего рода "виртуальный провод", который помогает разграничить передаваемые данные по своим маршрутам. К примеру, видео-поток в skype, если игнорировать роль порта, может передаваться в браузер, и наоборот, веб=страница будет отображаться в skype. Красноречивой иллюстрацией работы порта может служить альтернатива в виде провода, служащего для передачи пакета данных для отдельной программы. И этих программ и служб, разумеется, на одном устройстве может быть бесчисленное множество. 

Локальный ip-адрес может быть вида: 127.0.0.1
Непосредственно после него может быть записан номер порта: 127.0.0.1:80 или 127.0.0.1:443, 127.0.0.1:3000 и т.д.

Docker является виртуальным контейнером, например в нем присутствует контейнер того же xampp. То есть в таком случае, xampp не обязательно устанавливать на компьютер, можно реализовать его работу в контейнере. Docker при этом установит необходимую среду в самом минимальном наборе, например linux, и уже за тем на этой основе будет установлен xampp.
Docker часто становится актуален для больших проектов, там где требуется использование различных конфигураций. При этом вся среда разработки будет оставаться сугубо изолированной в отдельном контейнере. 

Работа с локальным сервером происходит по следующей схеме:
При обращении к адресу на локальном сервере, браузер обращается к системе с запросом для сопоставления ip-адреса с определенным портом и имени прописанного в браузере
Далее если прописывается "localhost", то обычно система сопоставляя по своим локальным правилам адрес находит совпадение с 127.0.0.1, данный адрес принадлежит самому компьютеру
На порту, например на 80-ом, будет работать apache. Система в свою очередь запрашивает у него нужную страницу и apache удовлетворяет запрос, так как он может генерировать только определенную страницу. 
Если же система не найдет localhost в локальных правилах, или ip-адрес был бы не найден там же, то запрос бы ушел на настоящий dns-сервер.

Существует "прослойка" локальных настроек, описанная выше. Она нужна как для разработки, так и для других целей.

При инициализации пустого git-репозитория необходимо в зависимости от последующих потребностей указать форму инициализации. Речь идет о глобальной или локальной опции. При выборе локальной формы указанные атрибуты пользователя, такие как email и имя будут определены только в определенной указанной папке. В случае с глобальной настройкой атрибуты будут применены и к остальным директориям.

Связать локальный репозиторий с удаленным возможно применив следующую команду:
git remote add origin (url удаленного репозитория)

Переход между ветками:
git checkout (имя ветки)

Перенос конфигурации с удаленного репозитория на локальный репозиторий:
git fetch

Обновление локального репозитория из удаленного репозитория 
git pull

Resource - специальная переменная содержащая ссылку на внешний ресурс

"Ресурсы" создаются и используются специальными функциями
Тип ресурса содержит указатели на открытые файлы соединения с базой данных, области изображений и т.д. 

Одинарные кавычки при использовании с переменными отображают переменные как текст, двойные кавычки же позволяют отобразить значение переменной, заключенной в эти кавычки.

В php можно явно указывать тип данных для аргументов, например для случаев когда неправильно используемый тип данных может привести к ошибке. Во избежание этого предоставляется возможность отклонять нежелательные аргументы для функции.

Можно передать в функцию вместе с третьим аргументом значение по умолчанию.

После всех аргументов,за скобками можно написать возвращаемый тип функции, напримен ":int". Это предполагает, что ожидается только указанный заранее тип данных.

Данный блок кода предназначен для программистов, для облегчения взаимодействия с проектом.

Если переданные значения в строковом виде будут содержать наряду с числами и другие символы, то будет вычисляться значение относящееся к числовому типу с самой первой позиции.
